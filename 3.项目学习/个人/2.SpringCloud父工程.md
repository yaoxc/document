
下面给出 **2025 版 IntelliJ IDEA** 中最简、最主流的 **“Spring Cloud 父工程”** 创建流程：  
采用 **Spring Boot 3.2 + Spring Cloud 2023.0.x**（cloud 版本对应关系官方已验证），**Maven 多模块**结构，一套操作直接生成可编译骨架。

---

### 一、前置检查
1. JDK 17+（Boot 3.x 硬性要求）
2. IDEA 2023.3+（内置 Spring Initializr 支持 Boot 3.2+）

---

### 二、创建父模块（仅 POM 聚合）
1. File → New → Project → **Spring Initializr**
    - **Group**: com.example
    - **Artifact**: sc-cloud-parent
    - **Type**: Maven
    - **Java Version**: 17
    - **Packaging**: **POM**  ⭐（关键：选 POM 就是父工程）
    - **Boot Version**: 3.2.8（2025-06 最新稳定版）
    - **Dependencies**: 暂时不勾，父模块只放统一依赖管理  
      → **Finish**

2. 删掉 src 目录（父工程不需要源码）

---

### 三、在父 pom 里统一 Spring Cloud 版本
打开 `pom.xml`，留最精简骨架 + cloud 依赖管理：

```xml
<project ...>
  <modelVersion>4.0.0</modelVersion>
  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.8</version>
    <relativePath/>
  </parent>

  <groupId>com.example</groupId>
  <artifactId>sc-cloud-parent</artifactId>
  <version>1.0-SNAPSHOT</version>
  <packaging>pom</packaging>

  <properties>
    <java.version>17</java.version>
    <!-- 2023.0.x 对应 Boot 3.2 -->
    <spring-cloud.version>2023.0.3</spring-cloud.version>
  </properties>

  <!-- 统一依赖管理 -->
  <dependencyManagement>
    <dependencies>
      <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-dependencies</artifactId>
        <version>${spring-cloud.version}</version>
        <type>pom</type>
        <scope>import</scope>
      </dependency>
    </dependencies>
  </dependencyManagement>

  <modules>
    <!-- 后续子模块在这里加 -->
  </modules>
</project>
```


**结论**：  
IDEA 2025 版直接 **Initializr 选 POM** → **父 pom 里 import cloud-dependencies** → **子模块选 Jar + 对应 Cloud 组件**，3 步即可得到官方推荐的“Spring Cloud 父工程”多模块骨架，零命令行、零 XML 手写。
