# docker启动报错

##  1. failed to solve: maven:3.8.3-jdk-8: failed to resolve source metadata for docker.io/library/maven:3.8.3-jdk-8
failed to solve: maven:3.8.3-jdk-8: failed to resolve source metadata for docker.io/library/maven:3.8.3-jdk-8: failed to do request: Head "https://registry.docker-cn.com/v2/library/maven/manifests/3.8.3-jdk-8?ns=docker.io": EOF

错误核心  
Docker 去 **国内镜像仓库 `docker.io` 的反向代理地址**（`registry.docker-cn.com`）拉取 `maven:3.8.3-jdk-8` 时直接 **EOF** ——  
该域名 2022 年起已 **彻底下线**，任何指向它的 `registry-mirrors` 都会导致 `Head ... EOF`。

------------------------------------------------
快速修复：把 Daemon 配置里的 **失效国内加速器删掉** 或 **换成仍在运行的镜像站**。

1. 打开 Docker Desktop  
   `Settings → Docker Engine` 把 `"registry-mirrors"` 里带 `docker-cn.com` 的那一行删掉/替换。

2. 推荐可用加速器（2025-06 验证）

   最终示例：
   ```json
   {
     "builder": { "gc": { "defaultKeepStorage": "20GB", "enabled": true } },
     "experimental": false,
     "registry-mirrors": [
       "https://mirror.ccs.tencentyun.com"
     ]
   }
   ```

3. 应用并重启 Docker  
   `Apply & Restart`
------------------------------------------------
若仍失败，可临时直连官方 Hub 再排查网络：

```bash
# 完全清空 mirrors（退回官方）
"registry-mirrors": []

# 或者一次性指定官方地址
docker pull maven:3.8.3-jdk-8 --platform linux/amd64
```

------------------------------------------------
一句话总结  
**把 `registry.docker-cn.com` 从 `registry-mirrors` 中移除或替换**，再重启 Docker，镜像即可正常解析。
